## Context

Each card variant needs three marketplace IDs: TCGPlayer product ID (already in card files), CardMarket ID, and CardTrader ID. These must be accurate and grouped by finish (normal, holofoil, reverse holofoil, etc.).

Alternatives considered:
- TCGPlayer API: requires credentials and a partner agreement; not publicly accessible
- CardMarket API: requires OAuth; only gives CardMarket IDs, not the others
- Manual mapping: not scalable across thousands of cards

## Decision

Use the TCGTracking `/tcgapi/v1/{categoryId}/sets/{setId}` endpoint. Each product in the response carries `id` (= TCGPlayer product ID), `cardmarket_id`, `cardtrader_id`, `number` (collector number), and `finishes[]`. This gives all three IDs in a single request per set, no credentials required.

The TCGPlayer product ID already present in the card file is used as the anchor to validate the match.

## Consequences

- One HTTP request per set (not per card) — fast and polite
- Coverage limited to sets that TCGTracking indexes (English + some Japanese; older sets may be missing)
- `cardtrader_id` at the product level is used for non-detailed-variant cards; the full `cardtrader[]` array is used when building per-variant entries
- Sets not in TCGTracking are recorded as `missing-endpoint` in the report and skipped
