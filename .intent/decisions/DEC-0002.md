## Context

TCGTracking products must be linked to specific card files. There is no shared key between a TCGTracking product and a `.ts` card file path — the link must be inferred from the data.

## Decision

Match by collector number first: normalise both the TCGTracking `number` field and the card filename (strip extension, lowercase, strip leading zeros). Accept only exact matches. If collector number matching yields no result for a card, fall back to normalised card name comparison.

Name-matched cards are flagged in the report as requiring human review before `--apply` is safe to use.

## Consequences

- Collector number matching is reliable for standard sets (Base Set onward)
- Name fallback catches promos and older sets that lack printed numbers
- Ambiguous matches (multiple products → one card) are never written — recorded for manual resolution
- Wrong matches are possible on cards that share a name across finishes; the existing `thirdParty.tcgplayer` on the card is used as a sanity check against the matched product's TCGPlayer ID
