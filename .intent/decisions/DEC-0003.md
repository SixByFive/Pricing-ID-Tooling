## Context

This tool permanently rewrites card source files. A wrong match or a bug in the writer could corrupt thousands of files across a set. The migration is hard to reverse without git.

## Decision

All runs default to dry-run mode. The `--apply` flag must be passed explicitly to write any files. Every run (dry or apply) writes a JSON report to `var/reports/` covering: resolved cards, unmatched cards, ambiguous cards, orphan products, and name-matched cards flagged for review.

## Consequences

- Contributors can inspect the report before committing to a write
- CI can run in dry-run mode to validate a set without risk
- The `--set` flag scopes a run to one set at a time, reducing blast radius further
- A `--set` + dry-run workflow is the recommended approach before any `--apply`
